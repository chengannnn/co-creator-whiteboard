## Codebase Patterns
- Scene + HistoryManager are stored as refs (not React state) to avoid unnecessary re-renders; canvas redraw triggered directly after scene mutations
- onSceneMutate callback pattern: CanvasComponent calls scene methods, then invokes onSceneMutate which handles history.push() + WebSocket broadcast in App.tsx
- All canvas rendering uses scene.getElements() — never direct shape arrays
- Move/resize uses moveElementsRef<SceneElement[]> for intermediate state, batch-updated via scene.updateElement() on pointer-up
- shapeToElement bridge still needed for initial server sync (server sends legacy Shape format)
- types/shapes.ts is DELETED — ToolType and DEFAULT_STYLE migrated to types/element.ts
- Monorepo uses npm workspaces with packages/frontend and packages/backend
- Backend uses ESM modules (`"type": "module"`) with tsx for development
- Frontend uses Vite + React 19 + TypeScript strict mode
- ESLint 9 flat config (eslint.config.js) required for both packages
- Backend ESLint needs `typescript-eslint` (not just `globals.node`) to parse TypeScript generics like `Map<string, Set<WebSocket>>`
- Dev servers: frontend on port 5173, backend on port 3001
- Vite proxies `/api` and `/ws` to backend during development
- Pure utility functions used by `useCallback` may need `/* eslint-disable react-hooks/exhaustive-deps */` if they don't depend on component state — ESLint auto-fix wraps them in `useCallback` which can cause parse errors
- Room state is managed in-memory on backend using `Map<string, Set<WebSocket>>` with cleanup timers
- WebSocket connection established on room mount; roomId sent via `join_room` message
- Use `forwardRef` + `useImperativeHandle` to expose child component methods (e.g., canvas export) to parent components
- rough.js uses `strokeLineDash` (e.g., `[8, 6]`) for dashed strokes — NOT `strokeStyle`
- Move pure helper functions to module level for stable references that don't trigger re-renders in hook dependency arrays
- Vitest is used for unit testing with jsdom environment; install via `npm install --save-dev vitest jsdom` and add `"test": "vitest run"` to scripts
- HistoryManager pattern: push() saves current state AFTER each committed action; undo() pops last saved state and restores it; redo() pops from forward history
- structuredClone works correctly in jsdom for deep copying SceneElement arrays with nested points arrays
- Test helper functions like makeRect() should use module scope to avoid unused import warnings in tsconfig strict mode
